Retail Data Normalization API

Normalize one or more retailer payloads into a single canonical product schema. Send an array of inputs; each input has retailerId (e.g. amazon, walmart, ebay, target, bestbuy, costco, shopify, custom) and the raw retailer JSON as payload. Response includes results per source, a merged products array, and per-source errors. One invalid payload does not fail others. Max request size 25MB.

Why use this API?

Product data from multiple retailers or marketplaces has different schemas. Normalize for price comparison, analytics, and catalog management.

What the API does

POST /normalize with product/retail payload(s). Returns normalized product(s).

About this API

Retail Data Normalization & Comparison API

The Retail Data Normalization & Comparison API enables teams to standardize, compare, and analyze retail product data across multiple retailers (Amazon, Walmart, eBay, Target, Best Buy, Costco, Shopify, Google Shopping, Etsy, and more) without fetching the data itself.

Send one request with an `inputs` array—each element has `retailerId` and raw `payload`. Users provide raw retailer responses (often large, inconsistent JSON payloads), and the API returns:

  • A single merged array of normalized, vendor-agnostic product objects
  • Per-source results and errors (one bad payload does not fail the rest)
  • Comparative insights (price, availability, reviews)
  • Deterministic rankings (cheapest, best value)

The API is fully stateless, scalable to large payloads (up to 25MB per request), and does not require vendor accounts.

What This API Does

**Normalize** — Transform retailer-specific JSON into a canonical product format: identifiers (ASIN, UPC, SKU), title, brand, category, images, price, availability, fulfillment, seller, reviews, raw source metadata.

**Compare** — On normalized products: cheapest retailer and price, price range, best-reviewed retailer, rankings. Optional query intent: cheapest, best_reviewed, in_stock_only, exclude_third_party.

**Extensible** — Supported retailers include Amazon, Walmart, eBay, Target, Best Buy, Costco, Shopify, Google Shopping, Etsy, Newegg, Flipkart, and custom. Unknown retailers use a best-effort generic normalizer.

Key Features

  • **Single Schema** — One canonical format for all retailers; one request returns one merged product array
  • **Multi-Retailer** — Amazon, Walmart, eBay, Target, Best Buy, Costco, Shopify, Google Shopping, Etsy, custom
  • **Comparison Ready** — Price and review comparison, rankings
  • **Stateless & Scalable** — No storage, 25MB payload limit, partial success per source
  • **No Vendor Keys** — Operates on user-provided data only
