Calendar Event Normalization API

Normalize one or more calendar payloads into a single canonical event schema. Send an inputs array; each input has calendarId (e.g. google, outlook, apple, ical, calendly, acuity, ews, generic) and the raw calendar JSON as payload. Optional defaultTimezone (IANA) and parsingHints. Response includes results per source, a merged events array (canonical events with start/end in original + UTC, recurrence, attendees, provenance), and per-source errors. One invalid payload does not fail others. Max request size 25MB.

Why use this API?

Calendar APIs return different structures. Normalize events from Google Calendar, Outlook, Apple Calendar, etc. into one schema for storage, display, or sync.

What the API does

POST /normalize with calendar provider payload(s). Returns normalized event(s).

About this API

Who Should Use This API

Calendar and scheduling apps, integration platforms, and teams unifying events across providers.

Also Known As

Calendar event normalizer, calendar API unifier, unified calendar schema.

Calendar Event Normalization API

One API. Every calendar. One canonical event. Convert Google, Outlook, Apple, Calendly, Exchange, iCal, and other calendar/scheduling payloads into a single canonical event format. Send JSON (POST /normalize) or raw .ics text (POST /normalize/ical); get one merged events array with timezone-safe start/end (original + UTC), recurrence, attendees, organizer, provenance, and confidence.

What This API Does

  • **Normalizes** calendar and scheduling data from Google, Outlook, Apple, iCal, CalDAV, Yahoo, Zoho, Fastmail, Calendly, Acuity, Canvas, Exchange (EWS), and generic/custom payloads into one canonical event schema.
  • **Timezone-explicit** — Each event has start/end with original time, UTC, IANA timezone, offset, and floating-time flag.
  • **Recurrence-aware** — RRULE and recurrence exceptions (EXDATE, overridden/cancelled) normalized; no need to parse vendor-specific rules yourself.
  • **Partial success** — Per-source results and errors; one bad payload does not fail the rest. Stateless, no outbound calls, 25MB max per request.

**One schema, one response.** Build unified calendar views, scheduling dashboards, CRM timelines, or analytics pipelines without per-provider parsing or timezone/recurrence edge cases.

Key Features

  • **Single schema** — One canonical event format for all sources; one request returns one merged events array.
  • **Multi-source** — Google, Outlook, Apple, iCal, CalDAV, Yahoo, Zoho, Fastmail, Calendly, Acuity, Canvas, Exchange (EWS), generic.
  • **Timezone-explicit** — Original time, UTC, IANA timezone, offset, floating flag in every event.
  • **Recurrence** — Normalized frequency, interval, byDay, count, until, rrule; recurrence exceptions supported.
  • **Stateless & scalable** — No storage, no vendor keys; 25MB per request; partial success per source.

Use Cases

  • **Unified calendar apps** — Show Google, Outlook, Apple events in one view with one schema.
  • **Scheduling & booking** — Normalize Calendly, Acuity, custom API events for dashboards or overlap detection.
  • **CRM & support** — Ingest Exchange, Fastmail, Zoho, Google into one timeline.
  • **Analytics & BI** — Ingest iCal feeds, Canvas, EWS into one schema for capacity or meeting density.
  • **Platform & product teams** — Reduce per-provider parsing and timezone/recurrence logic.
