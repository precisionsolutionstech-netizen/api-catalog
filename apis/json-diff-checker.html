JSON Diff Checker API

JSON diff checker, comparator, and change detector API. Compare two JSON payloads for breaking and non-breaking changes. API versioning, backward compatibility, CI/CD. Rule-based, deterministic. API contract testing, regression detection.

Why use this API?

API responses change over time. This API detects breaking changes (removed fields, type changes) and non-breaking changes so you can safely evolve APIs or fail CI when contracts break.

What the API does

POST /diff with before and after JSON. Returns classified changes (breaking vs non-breaking).

About this API

JSON Diff Checker API

The JSON Diff Checker API detects breaking and non-breaking changes between two JSON payloads. It is designed for developers who need to safely evolve APIs, validate backward compatibility, and catch schema-breaking changes before they reach production.

Unlike simple text diffs or shallow comparisons, this API understands JSON structure, data types, nullability, and array schemas, making it suitable for CI/CD pipelines, API versioning workflows, and automated regression checks.

What This API Does

Given a `before` and `after` payload, this API detects:

**Breaking changes:** Removed fields, field type changes, root type changes, nullability changes (non-nullable → nullable), structural changes inside arrays, nested object changes.

**Non-breaking changes:** Newly added fields, nullable → non-nullable transitions.

Each change is reported with a clear classification, machine-readable change code, and dot-notated field path.

Key Features

  • **Deterministic & Rule-Based** — Same input always yields the same output
  • **Deep Structural Comparison** — At any nesting level
  • **10MB Max per Payload** — Stateless, no data stored
  • **Production-Ready** — Ideal for API evolution, versioning, and regression checks

Use Cases

  • **API Versioning** — Validate that new versions don’t break existing consumers
  • **CI/CD Pipelines** — Fail builds when breaking changes are introduced
  • **Backward Compatibility** — Compare contract changes before release
  • **Regression Testing** — Automate structural diff checks
