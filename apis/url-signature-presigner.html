URL Signature Presigner API

Presigned URL generator, signer, and secure link API. Create cryptographically signed URLs for temporary downloads, uploads, callbacks, and CDN content. HMAC-SHA256, IP restrictions, method binding. Stateless, no database required.

Why use this API?

Grant temporary access to downloads, uploads, or callbacks without databases or sessions. HMAC-SHA256 signatures; works with any CDN or server.

What the API does

POST /sign with URL and constraints (expiry, method, IP, etc.). Returns signed URL. POST /verify to verify a request.

About this API

URL Signature Presigner API

The URL Signature Presigner API enables teams to securely grant temporary access to URLs (downloads, uploads, redirects, callbacks) using cryptographically signed URLs, without requiring databases, sessions, or persistent state. All authorization logic is encoded in the URL signature itself. Verification is fully deterministic and stateless.

What This API Does

Given a URL and access constraints, this API generates a cryptographically signed URL that:

  • **Encodes expiration time** — URL automatically becomes invalid after expiration
  • **Binds to HTTP methods** — Restrict to GET, POST, PUT, DELETE, etc.
  • **Restricts by IP address** — Allow only specific IPs or CIDR ranges
  • **Requires headers** — Enforce Content-Type, Authorization, or custom headers
  • **Limits payload size** — Prevent oversized requests
  • **Includes custom metadata** — Embed opaque, signed metadata for your use case

The signed URL can be verified without server-side state, making it perfect for CDNs, edge workers, and stateless architectures.

Key Features

  • **Stateless & Secure** — No databases or sessions required; authorization encoded in URL signature
  • **Vendor-Agnostic** — Works with any CDN, edge worker, or server; no vendor lock-in
  • **Deterministic Verification** — Identical inputs produce identical outputs; fully debuggable
  • **Production-Ready** — Fast enough for edge usage; timing-safe comparisons; clock skew handling
  • **Flexible Constraints** — Restrict by time, HTTP method, IP address, headers, or payload size
  • **Cryptographically Secure** — HMAC-SHA256/SHA512 signatures; tamper-proof URLs

Use Cases

  • **Temporary File Downloads** — Generate secure download links that expire automatically
  • **Secure Upload URLs** — Allow direct uploads to storage with size and time restrictions
  • **One-Time Report Access** — Generate links to sensitive reports with IP and time restrictions
  • **Callback URLs with Limited Validity** — Issue webhook callback URLs valid only for specific time windows
  • **CDN-Backed Protected Content** — Serve protected content via CDN without origin authentication
  • **API Gateway Temporary Access** — Grant temporary API access tokens encoded in URLs
  • **Presigned URLs** — Generate presigned URLs for S3, Cloud Storage, and similar services
  • **Stateless Authentication** — Implement stateless access control without databases

Who This Is For

### Backend Engineers

Generate secure presigned URLs for file downloads, uploads, and temporary access without building custom cryptographic logic.

### Platform / Infra Teams

Standardize URL signing across services. Ensure consistent, secure temporary access patterns.

### Edge / CDN Developers

Verify presigned URLs at the edge without database lookups. Perfect for Cloudflare Workers, AWS Lambda@Edge, and similar platforms.

### SaaS Teams

Expose temporary download links, secure upload URLs, and one-time report access without maintaining session state.

### API Providers

Issue callback URLs and webhook endpoints with limited validity. Control access by time, method, and source.
