JSON Schema Validator API

JSON schema validator, checker, and contract enforcer API. Validate JSON against structural schemas. Required fields, types, nullability. Strict, deterministic. CI/CD, API contract validation, data ingestion. Schema via URL or inline.

Why use this API?

Ensure request or response payloads conform to a schema before processing or storing.

What the API does

POST with JSON and schema. Returns validation result and errors.

About this API

JSON Schema Validator API

The JSON Schema Validator API provides strict, deterministic validation of JSON payloads against structural schemas. It is designed for developers who need reliable contract enforcement in CI/CD pipelines, backend services, and data ingestion workflows—without the complexity or ambiguity of the full JSON Schema specification.

What This API Does

Given a schema and a payload, this API:

  • Enforces required vs optional fields
  • Validates exact data types
  • Enforces nullability rules
  • Validates nested objects at any depth
  • Validates arrays and array item schemas
  • Detects extra fields not defined in the schema
  • Returns all validation errors in one deterministic response

**No type coercion. No guesswork. No fail-fast behavior.**

Schema can be provided inline (`schema`) or via URL (`schemaUrl`). Only HTTP/HTTPS URLs are allowed; private IPs and localhost are blocked.

Key Features

  • **Strict by Design** — Extra fields reported as errors; no coercion
  • **Deterministic** — Same input always yields same output
  • **Schema via URL** — Fetch schema from HTTPS (max 1MB, 5s timeout)
  • **10MB Payload Max, 100 Nesting Levels** — Stateless, no data stored

Use Cases

  • **Contract Enforcement** — Validate API requests/responses against schemas
  • **CI/CD** — Fail builds when payloads violate schema
  • **Data Ingestion** — Verify incoming JSON before processing
  • **Backend Services** — Enforce structural contracts without full JSON Schema
